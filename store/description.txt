===== DETAILED DESCRIPTION =====

VibeDesign

See a website you love? Extract its exact design DNA and turn it into a prompt that AI coding tools can reproduce faithfully.

WHAT IT DOES

Analyzes any website's visual design: colors, typography, shadows, spacing, animations, components
Generates narrative design briefs optimized for v0.dev, Bolt, Lovable, and general AI tools
Captures the design intent, not just CSS values — prompts explain WHY each choice works
Smart visual context inference: detects hero sections, scroll-reveal content, masonry grids, custom cursors, rotating text, and more

AI-ENHANCED (Optional)
Connect your own API key (Gemini, Claude, or OpenAI) for AI-powered design direction with 8-paragraph narrative briefs covering brand personality, visual hierarchy, typography, color system, interaction paradigm, and hover choreography. Works perfectly without AI too — the built-in rule engine covers all design patterns.

DESIGN SYSTEM EXPORT
Generate a reusable design system reference with derivation rules. Build pricing pages, docs, settings — any sub-page that stays true to the original design language.

PRIVACY-FIRST
Zero data collection, zero tracking, zero analytics
API keys stored locally, never transmitted to us
All processing happens in your browser
Select "None" as AI provider for zero network requests

HOW TO USE
1. Navigate to any website
2. Click the VibeDesign icon or open the side panel
3. Choose your focus (All, Colors, Type, Shadow, Motion, Layout, Components)
4. Click "Analyze Page" — your prompt is ready to paste into v0.dev, Bolt, or Lovable

WHAT'S NEW IN 1.5.3

Design Brief Summary — every prompt now starts with a 3-word brand personality snapshot, visual strategy, motion philosophy, and design constraints
Interaction Paradigm section — describes whether the site is scroll-driven or click-driven, hover feedback philosophy, and timing intent
Color semantic role matrix — maps extracted colors to actionable roles: primary-action, surface, border, text-primary, text-secondary, disabled
Typography semantic roles — each font level (H1-H4, body, caption, code) includes its usage context and intent
Visual classification — detects whether visuals are photography, illustration, abstract/geometric, video, or 3D
Card style extraction — captures card padding, radius, shadow system, border, and hover effects
CSS filter and blend mode detection — identifies grayscale, saturation, blur, and glassmorphism effects
Shadow system analysis — classifies shadows as elevation, glow, brutalist, or inset with depth scoring
Section eyebrow labels — captures overline text like "OUR APPROACH" per section
Numbered step content — captures each step's label and heading (STEP 01, STEP 02...)
Improved canvas animation scoping — prevents background animations from leaking into wrong sections
Global Tokens tab fix — now correctly shows the design system output
Increased extraction depth — more colors, shadows, animations, and transitions captured from complex sites
Enhanced AI prompts — richer context sent to AI providers for more precise design direction
52 silent error catches replaced with debug logging for better developer diagnostics
Full accessibility upgrade — aria-labels, semantic roles, screen reader support across all UI
